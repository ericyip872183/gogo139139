# 开发规范

> 这些规则是强制执行的，不是建议。

## 最高优先级：先给方案，确认后执行

每次会话，无论什么任务，必须先提供完整方案，等用户确认后再开始执行。

**授权后可自主执行（无需再确认）**：创建/修改/删除文件、编写代码、自测、更新文档、清理临时文件

**需要额外确认**：数据库迁移、Git提交、部署操作、删除大量文件

## TypeScript 规范

- 禁止使用 `any` 类型，严格模式
- 所有函数和接口必须有类型定义
- 优先用 `interface` 而非 `type` 定义对象

## 多租户隔离（核心）

- 所有数据库查询必须带 `tenantId`
- `tenantId` 从 JWT Token 中提取，**不从请求参数传入**
- JWT Payload 结构：`{ sub: userId, tenantId, role }`

```typescript
// 正确做法：从 @CurrentUser() 装饰器获取
@Get()
findAll(@CurrentUser() user: JwtPayload) {
  return this.service.findAll(user.tenantId)
}
```

## 文件头部注释

每个新建文件顶部必须有：

```typescript
/**
 * 文件：xxx.service.ts
 * 说明：XXX 管理服务（数据库操作）
 * 权限：TEACHER+
 * 对应接口：GET/POST/PATCH/DELETE /api/xxx
 */
```

## 后端响应格式

统一由拦截器处理，业务代码直接 return 数据即可：

```json
// 成功
{ "code": 200, "message": "success", "data": {}, "timestamp": "..." }

// 分页
{ "total": 100, "list": [], "page": 1, "pageSize": 20 }

// 错误（由 GlobalExceptionFilter 处理）
{ "code": 400, "message": "错误信息", "data": null, "timestamp": "..." }
```

## 后端配置

- 全局前缀：`/api`
- CORS：已启用（origin: true, credentials: true）
- 静态文件：`/uploads` 目录
- ValidationPipe：`whitelist: true, transform: true, forbidNonWhitelisted: true`
- 端口：`process.env.APP_PORT ?? 3000`

## 权限控制

```typescript
// 使用 @Roles() 装饰器 + RolesGuard
@Roles('TEACHER', 'TENANT_ADMIN')
@UseGuards(AuthGuard('jwt'), RolesGuard)
@Get()
findAll() {}
```

## 自测要求

功能完成后必须自己先测试，不让用户当测试员。测试后记录日志。

## 文档更新

完成功能后必须更新：
- `docs/PROGRESS.md` — 功能状态（❌→✅）
- `docs/API.md` — 新增接口
- `docs/DATABASE.md` — 新增表或字段

## 文件命名规范

- 后端：`xxx.module.ts` / `xxx.controller.ts` / `xxx.service.ts` / `xxx.dto.ts`
- 前端页面：`XxxView.vue`
- 前端API：`xxx.ts`（在 `src/api/` 下）
