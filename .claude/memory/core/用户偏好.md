# 用户偏好

## 协作方式

- 每次任务先给方案，确认后再执行（最高优先级，见开发规范）
- 文档内容使用中文
- 简洁直接，不废话，不用 emoji
- 重要事项单独列出说明
- 提交代码前必须先给用户确认，不擅自提交

## 沟通风格

- 回复要短，直接给结论
- 不要重复解释已经做过的事
- 不要在回复末尾做冗长总结
- 不要用 emoji 装饰文档

## 禁止事项

| 禁止 | 原因 |
|------|------|
| 使用 `any` 类型 | TypeScript 严格模式 |
| 缺少 tenantId 隔离 | 多租户安全要求 |
| 新文件无头部注释 | 代码可读性规范 |
| 跳过自测 | 不让用户当测试员 |
| 擅自提交代码 | 用户需要先审查 |
| 过多 emoji | 保持专业 |
| 冗长解释 | 用户时间宝贵 |

## 技术偏好

- 前端：Vue 3 + TypeScript + Element Plus
- 后端：NestJS + Prisma + MySQL
- 容器：Docker + Docker Compose
- 用 `interface` 而非 `type` 定义对象类型
