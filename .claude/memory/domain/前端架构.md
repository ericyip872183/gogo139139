# 前端架构

## 路由结构

两个布局：`MainLayout.vue`（主系统）和 `ModuleLayout.vue`（专业模块独立界面）

```
/login                      — 登录页（公开）
/                           — 主系统（需认证，MainLayout）
  ├─ /dashboard             — 首页
  ├─ /organizations         — 组织架构
  ├─ /users                 — 用户管理
  ├─ /questions             — 题库管理
  ├─ /papers                — 试卷管理
  ├─ /exams                 — 考试管理
  ├─ /exam/:id              — 在线考试（隐藏导航）
  ├─ /scores                — 成绩查询
  ├─ /score-tables          — 评分表管理
  ├─ /score-tables/:id/judge — 打分界面（隐藏导航）
  ├─ /admin                 — 超管后台（SUPER_ADMIN）
  ├─ /hardware              — 硬件调试（SUPER_ADMIN）
  ├─ /ai-platform           — AI平台管理（SUPER_ADMIN）
  ├─ /ai-service            — AI服务中心（SUPER_ADMIN/TENANT_ADMIN）
  ├─ /my-exams              — 学生考试列表
  └─ /profile               — 个人中心
/module/:code               — 专业模块（ModuleLayout）
```

路由守卫：`meta.requiresAuth`（认证检查）+ `meta.roles`（角色白名单）

## 菜单权限

| 角色 | 可见菜单 |
|------|----------|
| SUPER_ADMIN | 超管后台 + 全部管理菜单 + AI平台管理 + 硬件调试 |
| TENANT_ADMIN | 组织/用户/题库/试卷/考试/评分表/AI服务 |
| CLASS_ADMIN | 同TENANT_ADMIN |
| TEACHER | 题库/试卷/考试/成绩/评分表 |
| STUDENT | 我的考试/成绩查询 |

## 状态管理（Pinia）

**文件**：`stores/auth.ts`

```typescript
// 状态
token: string | null
user: { id, username, realName, role, avatar, tenantId, tenantName }
modules: PurchasedModule[]

// 计算属性
isLoggedIn: boolean
isTeacher: boolean  // TEACHER/TENANT_ADMIN/CLASS_ADMIN/SUPER_ADMIN
isSuperAdmin: boolean
canManageUsers: boolean  // SUPER_ADMIN/TENANT_ADMIN/CLASS_ADMIN
```

存储：localStorage 持久化（token、user、modules）

## API层

**文件**：`api/request.ts`

两个 axios 实例：
- `request`：15秒超时（通用请求）
- `imageRequest`：120秒超时（图片生成）

请求拦截：自动携带 `Bearer Token`，清除空参数
响应拦截：统一错误处理，401 自动登出

**12个API模块**：

| 文件 | 对应功能 |
|------|----------|
| auth.ts | 登录、个人资料、已购模块 |
| users.ts | 用户CRUD、批量操作、Excel导入导出 |
| questions.ts | 题库、分类、媒体、AI导入 |
| exams.ts | 试卷+考试管理 |
| exam-room.ts | 在线答题 |
| scores.ts | 成绩查询 |
| score-tables.ts | 评分表 |
| organizations.ts | 组织架构 |
| ai.ts | AI用户功能（对话/OCR/充值） |
| ai-admin.ts | AI管理（服务商/模型/配额/统计） |
| admin.ts | 超管后台 |
| request.ts | 请求配置（不是业务API） |

## 构建配置

**Vite 5** 配置（`vite.config.ts`）：
- 开发代理：`/api` → `http://localhost:3000`，`/uploads` → `http://localhost:3000`
- 路径别名：`@` → `./src`
- 自动导入：Vue3 API / Vue Router / Pinia / Element Plus 组件

## 硬件通信模块

**路径**：`packages/hardware-bridge/src/`

- `core/` — 抽象接口（IHardwareDevice、IDataFrame）
- `serial/` — Web Serial API 实现
- `bluetooth/` — Web Bluetooth API 实现
- `protocol/` — 协议解析（FrameParser、FrameEncoder）
- `demo/` — 调试面板、模拟器

前端通过 `HardwareDebugPanel.vue` 调试，仅 SUPER_ADMIN 可访问。
