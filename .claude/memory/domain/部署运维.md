## 常用命令

```bash
# 本地开发
docker-compose up -d          # 启动MySQL+Redis
pnpm dev                      # 启动前后端

# 容器操作
docker restart panlei-backend   # 修改后端代码后
docker restart panlei-frontend  # 修改前端代码后
docker logs panlei-backend -f --tail 100  # 查看后端日志
docker ps                       # 查看容器状态

# 数据库
pnpm db:migrate               # 运行迁移
pnpm db:generate              # 生成Prisma client
pnpm db:studio                # Prisma Studio可视化

# 构建
pnpm --filter backend build   # 构建后端
```

## 提交并部署到服务器（完整流程）

**代码仓库**：https://github.com/ericyip872183/gogo139139.git

**无CI/CD，需手动SSH到服务器执行部署。**

### 第一步：本地提交并推送

```bash
# 查看变更
git status
git diff

# 暂存并提交（确认后执行）
git add <具体文件>
git commit -m "feat: xxx"

# 推送到 GitHub
git push origin master
```

### 第二步：SSH到服务器执行部署

```bash
# 连接服务器
ssh -i C:/Users/23791/.ssh/id_rsa root@120.77.144.5

# 进入项目目录
cd /root/panlei

# 拉取最新代码
git pull

# 执行部署脚本（重新构建镜像并启动）
chmod +x deploy/deploy.sh
./deploy/deploy.sh
```

### deploy.sh 做了什么

1. 检查 `.env` 文件是否存在
2. `git pull` 拉取最新代码
3. `docker compose build --no-cache` 重新构建镜像
4. `docker compose up -d` 启动所有容器
5. 等待数据库就绪（15秒）
6. `prisma migrate deploy` 执行数据库迁移

### 只改了后端代码（不需要重新构建）

```bash
# 服务器上
cd /root/panlei
git pull
docker restart panlei-backend
```

### 查看部署日志

```bash
docker compose -f docker-compose.prod.yml logs -f
# 或单独查看后端
docker logs panlei-backend -f --tail 100
```

## 重置管理员密码

```bash
docker exec panlei-backend node -e "
const { PrismaClient } = require('@prisma/client');
const bcrypt = require('bcrypt');
const prisma = new PrismaClient();
(async () => {
  const hash = await bcrypt.hash('admin123', 10);
  await prisma.user.updateMany({ where: { username: 'admin' }, data: { password: hash } });
  console.log('Done');
  await prisma.\$disconnect();
})();
"
```
