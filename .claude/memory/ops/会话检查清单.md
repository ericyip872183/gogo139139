# 会话检查清单

## 会话开始

1. 读 `MEMORY.md` — 了解记忆结构
2. 读 `core/项目概览.md` — 确认项目背景
3. 读 `docs/PROGRESS.md` — 了解当前功能状态
4. 确认本次任务范围，给出方案，等待用户确认

## 会话结束

1. 确认所有完成的功能已自测
2. 更新 `docs/PROGRESS.md`（❌→✅）
3. 如有新接口，更新 `docs/API.md`
4. 如有新表/字段，更新 `docs/DATABASE.md`
5. 提醒用户确认是否需要提交代码
6. 如有新发现的规律/问题，更新对应记忆文件

## 记忆文件更新时机

| 触发场景 | 更新文件 |
|----------|----------|
| 用户纠正做法 | core/用户偏好.md |
| 发现新的调试技巧 | ops/常见问题速查.md |
| 修复了新Bug | ops/Bug修复经验.md |
| 部署流程有变化 | domain/部署运维.md |
