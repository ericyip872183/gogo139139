## 第一部分：有多少 Docker 容器

### 1.1 本地电脑（你的电脑）

**本地运行的容器**（docker-compose.yml）：

```
你的电脑（c:\项目\panlei）
├─ mysql:8.0      ← MySQL 数据库（端口 3306）
├─ redis:7-alpine  ← Redis 缓存（端口 6379）
└─ 没有前端/后端容器！  ← 本地直接用 pnpm dev 运行
```

**本地开发模式**：
- 用 Docker 只跑 MySQL + Redis
- 前端后端直接在你电脑上运行（pnpm dev）
- **不是完整的 Docker 化**

---

### 1.2 服务器（120.77.144.5）

**服务器运行的容器**（docker-compose.prod.yml）：

```
服务器（120.77.144.5）
├─ panlei-mysql     ← MySQL 数据库（端口 3306）
├─ panlei-redis     ← Redis 缓存（端口 6379）
├─ panlei-backend   ← 后端服务（端口 3000）
└─ panlei-frontend  ← 前端 + Nginx（端口 80/443）
```

**服务器上实际运行的**（从你的服务器查到）：

| 容器名 | 状态 | 端口 | 功能 |
|--------|------|------|------|
| panlei-backend | Up 2 days | 3000/tcp | 后端 API 服务 |
| panlei-redis | Up 2 days | 6379/tcp | 缓存服务 |
| panlei-mysql | Up 2 days (healthy) | 3306/tcp | 数据库 |

## 第二部分：每个容器的功能（说人话）

### 2.1 panlei-mysql（数据库）

**比喻**：仓库货架

```
功能：
├─ 存储所有数据：用户/题目/试卷/成绩
├─ 像 Excel 表格一样，有行有列
└─ 数据永久保存，容器删了数据还在

技术细节：
- 镜像：mysql:8.0
- 数据卷：mysql_data（数据不会丢失）
- 端口：3306（容器内部，不对外暴露）
```

**人话**：如果没有数据库，就像开餐厅没有仓库，食材放哪里？

---

### 2.2 panlei-redis（缓存）

**比喻**：前台快取柜

```
功能：
├─ 临时存储常用数据（比如登录 Token）
├─ 比 MySQL 快 100 倍
└─ 数据可以丢失（重启清空没关系）

技术细节：
- 镜像：redis:7-alpine
- 数据卷：redis_data
- 端口：6379（容器内部，不对外暴露）
```

**人话**：餐厅客人多的时候，把常用餐具放前台，不用每次去仓库拿。

---

### 2.3 panlei-backend（后端）

**比喻**：后厨

```
功能：
├─ 处理所有业务逻辑（登录/出题/考试）
├─ 读写 MySQL 和 Redis
├─ 提供 API 接口给前端调用
└─ 端口 3000（容器内部，不对外暴露）

技术细节：
- 镜像：自己构建（packages/backend/Dockerfile）
- 依赖：必须等 MySQL 和 Redis 启动
- 自动重启：挂了会自动重启
```

**人话**：客人点单，后厨做菜，做好端上来。

---

### 2.4 panlei-frontend（前端 + Nginx）

**比喻**：前厅 + 门卫

```
功能：
├─ 1. 提供前端页面（HTML/CSS/JS）
├─ 2. 门卫：处理 HTTPS 加密
├─ 3. 调度：/api/ 请求转发给后端
├─ 4. 对外暴露端口 80（HTTP）和 443（HTTPS）
└─ 这是用户能直接访问的入口！

技术细节：
- 镜像：自己构建（packages/frontend/Dockerfile）
- 内置：Nginx 服务器
- SSL 证书：/etc/nginx/ssl/
```

**人话**：客人看到的餐厅门面，也是客人唯一能进去的门。

## 第四部分：是否可以独立封装

### 4.1 答案：是的，全部都是独立的

**每个容器都是独立的集装箱**：

```
┌─────────────────────────────────────────┐
│  panlei-mysql 集装箱                    │
│  - 只有 MySQL                          │
│  - 可以单独启动/停止                   │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  panlei-redis 集装箱                    │
│  - 只有 Redis                          │
│  - 可以单独启动/停止                   │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  panlei-backend 集装箱                  │
│  - 只有后端代码 + Node.js              │
│  - 可以单独启动/停止                   │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  panlei-frontend 集装箱                 │
│  - 只有前端 + Nginx                    │
│  - 可以单独启动/停止                   │
└─────────────────────────────────────────┘
```

---

### 4.2 Docker 独立的好处

| 好处 | 说人话 |
|------|--------|
| 坏了不影响别的 | 后厨坏了，前厅还能展示菜单 |
| 可以单独升级 | 升级数据库，不需要重启前端 |
| 可以单独扩展 | 客人多了，多开几个后厨（后端） |
| 环境一致 | 你电脑的 MySQL = 服务器的 MySQL |

---

### 4.3 但是它们需要协作

**虽然独立，但它们之间需要通信**：

```
用户访问前端 → 前端问后端 → 后端查数据库 → 后端存缓存 → 返回给用户
```

**依赖关系**：
- `panlei-frontend` 依赖 `panlei-backend`（没有后端，前端没法干活）
- `panlei-backend` 依赖 `panlei-mysql` 和 `panlei-redis`（没有数据库和缓存，后端没法干活）

## 第五部分：为什么这个项目可以有网址，有的不行

### 5.1 网址访问需要什么？

**完整链条**：

```
1. 有公网 IP 地址
   ↓
2. 有域名（DNS 解析到 IP）
   ↓
3. 服务器有端口开放（80/443）
   ↓
4. 有 Nginx 处理请求
   ↓
5. 有 SSL 证书（HTTPS）
   ↓
6. 后端服务在运行
   ↓
✅ 可以访问！
```

---

### 5.2 这个项目为什么可以有网址

**因为满足了所有条件**：

| 条件 | 状态 | 说明 |
|------|------|------|
| 公网 IP | ✅ 有 | 120.77.144.5（阿里云服务器） |
| 域名 | ✅ 有 | pl.7025700.vip |
| 端口开放 | ✅ 有 | 80/443 端口已开放 |
| Nginx | ✅ 有 | panlei-frontend 容器里的 Nginx |
| SSL 证书 | ✅ 有 | Let's Encrypt 免费证书 |
| 服务运行 | ✅ 有 | 4 个容器都在运行 |

---

### 5.3 有的项目为什么不行

**常见原因**：

| 原因 | 说人话 | 解决方法 |
|------|--------|---------|
| 没有公网 IP | 服务器在你家，别人找不到 | 买阿里云/腾讯云服务器 |
| 没有域名 | 只有 IP，没有网址 | 买个域名（几十块钱一年） |
| 端口没开放 | 防火墙挡住了 | 在云服务器控制台开放 80/443 |
| 没有 Nginx | 直接跑后端，不安全 | 加 Nginx 做反向代理 |
| 没有 SSL 证书 | 浏览器提示不安全 | 申请 Let's Encrypt 免费证书 |
| 服务没跑起来 | 容器没启动 | docker-compose up -d |

---

### 5.4 用比喻理解网址访问

**比喻**：开餐厅

```
有公网 IP = 有商铺地址
有域名 = 有招牌名字（"若容平台"而不是"XX路XX号"）
端口开放 = 大门能进
Nginx = 门卫
SSL 证书 = 安全锁（客人放心进）
服务运行 = 餐厅在营业
```

**没有网址的餐厅**：
- 只有 IP = 只有商铺地址，没有招牌
- 端口没开放 = 大门锁着
- 没有 Nginx = 没有门卫，客人直接闯后厨
- 没有 SSL = 没有安全锁，客人不敢进

## 第六部分：整体观 — 从本地到服务器的完整链路

### 6.1 本地开发（你电脑）

```
你电脑
├─ Docker：只跑 MySQL + Redis
├─ 前端：直接运行 pnpm dev（端口 5173）
```

- 前端后端直接在你电脑上
- 方便调试，改代码立即生效

---

### 6.2 生产部署（服务器）

```
服务器（120.77.144.5）
├─ Docker：4 个容器都跑
│  ├─ panlei-mysql（数据库）
│  ├─ panlei-redis（缓存）
│  ├─ panlei-backend（后端，端口 3000）
│  └─ panlei-frontend（前端 + Nginx，端口 80/443）
└─ 访问：https://pl.7025700.vip
```

**特点**：
- 完整的 Docker 化
- 所有服务都在容器里
- 稳定、安全、可扩展

---

### 6.3 从本地到服务器的流程

```
本地开发
   ↓
改代码
   ↓
pnpm run build:frontend  ← 构建前端
   ↓
git add + commit + tag  ← 提交 Git
   ↓
./deploy/sync-to-server.sh  ← 上传到服务器
   ↓
服务器上重新构建容器
   ↓
docker-compose -f docker-compose.prod.yml up -d  ← 重启容器
   ↓
✅ 用户访问 https://pl.7025700.vip 看到新内容
```

## 第七部分：常用 Docker 命令

### 7.1 查看状态

```bash
# 查看所有容器
ssh root@120.77.144.5 "docker ps"

# 查看容器日志
ssh root@120.77.144.5 "docker logs panlei-backend --tail 50"

# 进入容器内部
ssh root@120.77.144.5 "docker exec -it panlei-mysql bash"
```

### 7.2 操作容器

```bash
# 重启所有容器
ssh root@120.77.144.5 "cd /root/panlei && docker-compose -f docker-compose.prod.yml restart"

# 只重启后端
ssh root@120.77.144.5 "docker restart panlei-backend"

# 停止所有容器
ssh root@120.77.144.5 "cd /root/panlei && docker-compose -f docker-compose.prod.yml down"

# 启动所有容器
ssh root@120.77.144.5 "cd /root/panlei && docker-compose -f docker-compose.prod.yml up -d"
```
