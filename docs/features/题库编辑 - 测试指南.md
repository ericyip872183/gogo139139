### 2.3 集成测试（使用 Postman）

#### 准备环境

在 Postman 中创建环境变量：
- `baseUrl`: `http://localhost:3000`
- `token`: 登录后自动填充

#### 测试流程

**Step 1: 登录获取 Token**

```http
POST {{baseUrl}}/api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123",
  "tenantCode": "PLATFORM"
}
```

响应处理脚本（自动保存 token）：
```javascript
if (pm.response.code === 200) {
  const json = pm.response.json();
  pm.environment.set('token', json.data.token);
}
```

**Step 2: 创建分类**

```http
POST {{baseUrl}}/api/questions/categories
Authorization: Bearer {{token}}
Content-Type: application/json

{
  "name": "测试分类",
  "sortOrder": 0
}
```

**Step 3: 创建题目**

```http
POST {{baseUrl}}/api/questions
Authorization: Bearer {{token}}
Content-Type: application/json

{
  "type": "SINGLE",
  "content": "<p>这是一道<b>测试题目</b></p>",
  "difficulty": "MEDIUM",
  "score": 2,
  "categoryId": "{{categoryId}}",
  "options": [
    { "label": "A", "content": "正确答案", "isCorrect": true },
    { "label": "B", "content": "错误答案", "isCorrect": false }
  ]
}
```

**Step 4: 上传媒体文件**

```http
POST {{baseUrl}}/api/questions/{{questionId}}/media
Authorization: Bearer {{token}}
Content-Type: multipart/form-data

file: [选择图片文件]
caption: 测试图片说明
```

**Step 5: 验证题目详情**

```http
GET {{baseUrl}}/api/questions/{{questionId}}
Authorization: Bearer {{token}}
```

验证响应包含 `mediaItems` 字段。

**Step 6: 删除媒体资源**

```http
DELETE {{baseUrl}}/api/questions/media/{{mediaId}}
Authorization: Bearer {{token}}
```

**Step 7: 删除题目**

```http
DELETE {{baseUrl}}/api/questions/{{questionId}}
Authorization: Bearer {{token}}
```

## 六、常见问题排查

### 问题 1：测试运行失败 "Cannot find module"

**解决：**
```bash
# 重新安装依赖
pnpm clean
pnpm install

# 重新构建
pnpm --filter backend build
```

### 问题 2：媒体上传后无法访问

**检查：**
1. `uploads` 目录是否存在
2. 后端 `main.ts` 是否配置静态文件服务
3. 文件路径是否正确包含 tenantId

### 问题 3：富文本编辑器不显示

**检查：**
1. TinyMCE 相关资源是否加载
2. 浏览器控制台是否有 CORS 错误
3. 尝试清除缓存

### 问题 4：测试覆盖率低

**提升建议：**
```bash
# 查看覆盖率报告
pnpm test -- --coverage

# 打开 HTML 报告
open coverage/index.html
```

  test:
