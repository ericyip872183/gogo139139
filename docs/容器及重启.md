# 盼蕾项目 - 容器与重启指南

> 本文档解释项目的容器架构、启动流程、以及常见问题的处理方法。

---

## 一、什么是容器？

### 1.1 简单理解

**容器 = 打包好的运行环境**

想象你要搬家：
- **传统方式**：把衣服、锅碗、家具全部拆散，带着所有零件到新家再组装
- **容器方式**：直接把整个房间打包（包括家具、日用品、甚至水电），到新家插电就能用

Docker 容器就是这样一种"打包"技术，它把**应用程序 + 依赖库 + 运行环境**全部打包，形成一个独立可运行的"盒子"。

### 1.2 容器 vs 传统安装

| 对比项 | 传统安装 | 容器方式 |
|--------|----------|----------|
| 安装难度 | 高（要配置各种环境） | 低（一行命令搞定） |
| 一致性 | 容易"在我电脑能跑"问题 | 哪里都能跑 |
| 隔离性 | 相互影响 | 互不干扰 |
| 卸载 | 各种残留 | 删除容器即可 |
| 资源 | 直接占用本机 | 按需分配 |

## 二、本项目有哪些容器？

### 2.1 架构图

```
┌─────────────────────────────────────────────────────────┐
│                      你的电脑                             │
│                                                         │
│   ┌─────────────┐    ┌─────────────┐    ┌───────────┐  │
│   │  前端容器    │    │  后端容器    │    │  数据库    │  │
│   │ panlei-     │    │ panlei-     │    │ 容器们    │  │
│   │ frontend    │◄──►│ backend     │◄──►│           │  │
│   │  :5173      │    │  :3000      │    │ mysql     │  │
│   └─────────────┘    └─────────────┘    │ :3306    │  │
│                                          │ redis    │  │
│   你访问的入口                            │ :6379    │  │
│   http://localhost:5173                  └───────────┘  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 四个容器介绍

| 容器名 | 作用 | 端口 | 内部地址 |
|--------|------|------|----------|
| **panlei-frontend** | 前端界面（Vue） | 5173 | - |
| **panlei-backend** | 后端服务（NestJS） | 3000 | - |
| **panlei-mysql** | 数据库 | 3306 | mysql:3306 |
| **panlei-redis** | 缓存/验证码存储 | 6379 | redis:6379 |

### 2.3 端口映射解释

- `3000:3000` 的意思是：**容器内3000端口 映射到 电脑的3000端口**
- 你在浏览器访问 `localhost:3000`，请求会转发到容器内的3000端口

### 2.4 容器间如何通信？

```
前端容器 ──(http://backend:3000)──► 后端容器 ──(mysql://mysql:3306)──► MySQL容器
                   │
                   └───────────────(redis://redis:6379)──► Redis容器
```

**重点**：容器之间用**容器名**（不是 localhost）来通信！
- 前端访问后端 → `http://backend:3000`
- 后端访问数据库 → `mysql:3306`
- 后端访问缓存 → `redis:6379`

## 三、为什么容器启动常有错误？

### 3.1 容器启动的本质

容器启动不是"打开一个软件"，而是**创建一个新的隔离环境**。这意味着：

1. **每次都是全新开始** - 之前手动修改的配置不会保留
2. **依赖网络** - 需要下载基础镜像、依赖包
3. **需要等待依赖服务** - 后端要等 MySQL 准备好才能启动

### 3.2 本项目常见错误及原因

#### 错误1：后端连不上 MySQL

```
PrismaClientInitializationError: Can't reach database server at `mysql:3306`
```

**原因**：
- MySQL 容器还没完全启动（健康检查还没通过）
- 容器网络没配好，后端找不到 mysql 容器

**解决**：
```bash
# 重启后端，让它等 MySQL 完全就绪后再连
docker restart panlei-backend
```

#### 错误2：前端 500 错误

```
GET /api/admin/dashboard/stats 500 (Internal Server Error)
```

**原因**：
- 前端容器内的代理配置指向 `localhost:3000`
- 但容器内的 localhost 指的是**前端容器自己**，不是后端容器

**解决**：
- 修改 vite.config.ts 的代理配置为 `http://backend:3000`
- 重新构建容器

#### 错误3：登录失败（密码错误）

```
{"code":401,"message":"用户名或密码错误"}
```

**原因**：
- 数据库里的密码 hash 是之前环境生成的
- 不同机器/环境下 bcrypt 生成的 hash 可能不一致

**解决**：
```bash
# 在后端容器内重新生成密码
docker exec panlei-backend node -e "
const { PrismaClient } = require('@prisma/client');
const bcrypt = require('bcrypt');
const prisma = new PrismaClient();
(async () => {
  const hash = await bcrypt.hash('admin123', 10);
  await prisma.\$connect();
  await prisma.user.updateMany({
    where: { username: 'admin' },
    data: { password: hash }
  });
  console.log('Password updated');
  await prisma.\$disconnect();
})();
"
```

## 四、启动流程（完整步骤）

### 4.1 首次启动

```bash
# 1. 确保 Docker Desktop 已启动（Windows 下需要打开 Docker Desktop 应用）

# 2. 进入项目目录
cd /c/项目/panlei

# 3. 启动所有容器
docker-compose -f docker-compose.dev.yml up -d

# 4. 等待服务就绪（约10-20秒）
# 检查状态
docker ps
```

### 4.2 日常重启

```bash
# 重启所有容器
docker-compose -f docker-compose.dev.yml restart

# 或者单独重启某个容器
docker restart panlei-backend   # 只重启后端
docker restart panlei-frontend  # 只重启前端
```

### 4.3 修改代码后

如果你修改了前端/后端代码，需要重新构建：

```bash
# 重新构建并启动
docker-compose -f docker-compose.dev.yml up -d --build

# 或者只重新构建前端/后端
docker-compose -f docker-compose.dev.yml up -d --build frontend
docker-compose -f docker-compose.dev.yml up -d --build backend
```

### 4.4 完全重置（删除所有数据）

⚠️ **警告：这会删除所有数据！**

```bash
# 1. 停止并删除所有容器
docker-compose -f docker-compose.dev.yml down

# 2. 删除数据卷（数据库数据会丢失）
docker volume rm panlei_mysql_data
docker volume rm panlei_redis_data

# 3. 重新启动（会重新初始化数据库）
docker-compose -f docker-compose.dev.yml up -d
```

## 五、常用命令汇总

### 5.1 查看状态

```bash
# 查看运行中的容器
docker ps

# 查看所有容器（包括停止的）
docker ps -a

# 查看容器日志
docker logs panlei-backend
docker logs panlei-frontend
docker logs panlei-mysql
docker logs panlei-redis

# 实时查看日志（类似 tail -f）
docker logs -f panlei-backend
```

### 5.2 容器操作

```bash
# 启动容器
docker start panlei-backend

# 停止容器
docker stop panlei-backend

# 重启容器
docker restart panlei-backend

# 进入容器内部（类似 SSH）
docker exec -it panlei-backend sh
docker exec -it panlei-frontend sh

# 查看容器内部配置
docker exec panlei-frontend cat /app/packages/frontend/vite.config.ts
```

### 5.3 数据库操作

```bash
# 进入 MySQL 容器
docker exec -it panlei-mysql mysql -upanlei -ppanlei_dev_2024 panlei_dev

# 执行 SQL
docker exec panlei-mysql mysql -upanlei -ppanlei_dev_2024 panlei_dev -e "SELECT * FROM users;"

# 初始化数据库（运行 seed 脚本）
docker exec panlei-backend npx prisma db push --skip-generate
```

## 六、调试技巧

### 6.1 前端调试

1. 打开浏览器访问 http://localhost:5173
2. 按 F12 打开开发者工具
3. 查看 Network 标签页的请求
4. 看到错误看 Console 标签页

### 6.2 后端调试

```bash
# 查看后端实时日志
docker logs -f panlei-backend

# 测试 API（需要先登录获取 token）
TOKEN="你的token"
curl -H "Authorization: Bearer $TOKEN" http://localhost:3000/api/admin/dashboard/stats
```

### 6.3 网络连通性测试

```bash
# 测试后端能否访问 MySQL
docker exec panlei-backend sh -c "nc -zv mysql 3306"

# 测试前端能否访问后端
docker exec panlei-frontend sh -c "nc -zv backend 3000"
```

## 七、常见问题速查

| 问题 | 解决 |
|------|------|
| Docker Desktop 启动不了 | 确保 BIOS 里开启了虚拟化，Windows 需要 WSL2 |
| 容器连不上 | `docker-compose down && docker-compose up -d` 重启 |
| 前端 500 错误 | 检查后端日志，确认后端正常运行 |
| 后端连不上数据库 | `docker restart panlei-backend` 重启后端 |
| 登录不上 | 重新执行密码初始化命令（见上文） |
| 修改代码没生效 | 需要 `--build` 重新构建 |

---

## 八、总结

1. **本项目有4个容器**：前端、后端、MySQL、Redis
2. **容器之间用容器名通信**：前端访问后端用 `http://backend:3000`
3. **重启命令**：`docker-compose -f docker-compose.dev.yml restart`
4. **修改代码后要重建**：`docker-compose -f docker-compose.dev.yml up -d --build`
5. **常见问题**：主要是网络配置和初始化问题，按本文档步骤处理即可
