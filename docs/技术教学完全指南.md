## 三、前端技术深入

### 3.1 Vue 3 组合式 API vs Options API

```
┌───────────────────────────────────────────────────────────────────────────┐
│                        Vue 2 Options API                                  │
├───────────────────────────────────────────────────────────────────────────┤
│  export default {                                                         │
│    data() {         // 数据在这里定义                                      │
│      return { count: 0 }                                                  │
│    },                                                                     │
│    methods() {      // 方法在那里定义                                     │
│      increment() {}                                                       │
│    },                                                                     │
│    computed() {     // 计算属性在另一个地方                               │
│      double() {}                                                          │
│    }                                                                      │
│  }                                                                        │
│                                                                           │
│  ❌ 问题：逻辑分散，复用困难                                              │
└───────────────────────────────────────────────────────────────────────────┘

┌───────────────────────────────────────────────────────────────────────────┐
│                        Vue 3 组合式 API                                   │
├───────────────────────────────────────────────────────────────────────────┤
│  <script setup lang="ts">                                                 │
│  import { ref, computed } from 'vue'                                      │
│                                                                           │
│  // 所有逻辑在一起，按功能组织                                             │
│  const count = ref(0)                                                     │
│  const double = computed(() => count.value * 2)                           │
│  const increment = () => count.value++                                    │
│  </script>                                                                │
│                                                                           │
│  ✅ 优势：逻辑聚合，易于复用，TS 友好                                     │
└───────────────────────────────────────────────────────────────────────────┘
```

### 3.2 响应式原理对比

```
╔═══════════════════════════════════════════════════════════════════════════╗
║                         Vue 2 vs Vue 3 响应式原理                          ║
╠═══════════════════════════════════════════════════════════════════════════╣
║                                                                           ║
║  Vue 2: Object.defineProperty()                                           ║
║  ┌─────────────────────────────────────────────────────────────────────┐ ║
║  │  Object.defineProperty(obj, 'count', {                              │ ║
║  │    get() { return count },      // 读取时收集依赖                   │ ║
║  │    set(val) { count = val; notify() }  // 更新时通知                │ ║
║  │  })                                                                 │ ║
║  │                                                                     │ ║
║  │  ⚠️ 限制：无法检测数组变化、无法检测对象新增属性                    │ ║
║  └─────────────────────────────────────────────────────────────────────┘ ║
║                                                                           ║
║  Vue 3: Proxy                                                             ║
║  ┌─────────────────────────────────────────────────────────────────────┐ ║
║  │  const reactive = new Proxy(obj, {                                  │ ║
║  │    get(target, key) { return target[key] },      // 拦截读取        │ ║
║  │    set(target, key, val) { target[key] = val; notify() } // 拦截写入│ ║
║  │  })                                                                 │ ║
║  │                                                                     │ ║
║  │  ✅ 优势：可以监听数组、可以监听新增/删除、性能更好                 │ ║
║  └─────────────────────────────────────────────────────────────────────┘ ║
║                                                                           ║
╚═══════════════════════════════════════════════════════════════════════════╝
```

### 4.2 依赖注入原理

```
╔═══════════════════════════════════════════════════════════════════════════╗
║                     传统方式 vs 依赖注入                                   ║
╠═══════════════════════════════════════════════════════════════════════════╣
║                                                                           ║
║  ❌ 传统方式：自己创建依赖                                                ║
║  ┌─────────────────────────────────────────────────────────────────────┐ ║
║  │  class UserService {                                                │ ║
║  │    private db = new Database()  // 硬编码，难以替换                 │ ║
║  │    private cache = new Redis()  // 耦合严重                         │ ║
║  │  }                                                                  │ ║
║  │                                                                     │ ║
║  │  问题：难以测试、难以维护、难以替换                                 │ ║
║  └─────────────────────────────────────────────────────────────────────┘ ║
║                                                                           ║
║  ✅ 依赖注入：别人给依赖                                                  ║
║  ┌─────────────────────────────────────────────────────────────────────┐ ║
║  │  @Injectable()                                                      │ ║
```
