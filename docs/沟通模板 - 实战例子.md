## 场景二：修复 Bug（登录失效问题）

### 会话开始（你这样说）

```markdown
## 会话目标
排查并修复：用户反馈登录提示「登录已失效，请重新登录」

- 版本：v1.1.1
- 现象：浏览器登录返回 401，但 curl 测试正常
- 已尝试：清除浏览器缓存、无痕模式，问题依旧

## 排查步骤
1. 检查服务器 index.html 是否最新
2. 检查后端 API 是否正常响应
3. 检查 JWT token 生成和验证
4. 检查 nginx CORS 配置
```

#### AI 开始排查后，你配合测试（这样说）


## 总结：沟通模板核心结构

```
1. 会话目标（一句话说清要做什么）
2. 当前状态（上次完成到哪里）
3. 任务拆解（分成几个小任务，一次一个）
4. 任务开始（涉及文件 + 步骤）
5. 任务完成（自测结果 + 提交 Git）
6. 会话结束（更新文档 + 下次起点）
```

**记住**：每次只说一件事，做完一件事再说下一件。
