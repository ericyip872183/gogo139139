## 如何分配新端口

**当你需要部署新项目时**：

1. 查已占用端口：`ssh root@IP "netstat -tlnp \| grep LISTEN"`
2. 查 PM2 进程：`ssh root@IP "pm2 list"`
3. 查 Docker 容器：`ssh root@IP "docker ps"`
4. 从上表预留端口中选择一个未被使用的
5. 更新此表格，标记为「已上线」

## Nginx 转发配置

**所有外部访问统一通过 Nginx 转发**：

```nginx
# 若容平台
server {
    listen 80;
    server_name pl.7025700.vip;

    location / {
        proxy_pass http://localhost:3002;  # 转发到后端
    }

    location /static/ {
        root /var/www/panlei;  # 静态文件
    }
}

# 新项目示例
server {
    listen 80;
    server_name new.7025700.vip;

    location / {
        proxy_pass http://localhost:3003;  # 转发到后端 3003
    }
}
```

## 端口冲突排查命令

```bash
# 查某个端口被谁占用
ssh root@IP "netstat -tlnp | grep :3002"

# 查某个进程的所有端口
ssh root@IP "netstat -tlnp | grep <PID>"

# 杀掉占用端口的进程
ssh root@IP "kill -9 <PID>"

# 重启 PM2 服务
ssh root@IP "pm2 restart <name>"

# 重启 Docker 容器
ssh root@IP "docker-compose restart"
```
